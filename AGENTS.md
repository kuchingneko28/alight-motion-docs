# Agent Instructions

## Key Gotchas & Code Generation

- **Do not edit generated files directly:**
  - `docs/index.md`, `docs/.vitepress/config.ts`, and markdown files under `docs/effects/`, `docs/shapes/`, `docs/blend-modes/`, `docs/transitions/`, and `docs/elements/` are autogenerated from XML configurations.
  - To modify their structure or contents, edit `scripts/generate-docs.ts` or files in `scripts/lib/` instead.
  - Only `docs/guide.md` is fully safe to edit directly.

- **Source of truth:**
  - Raw data is extracted from the `decompiled_apk/` directory:
    - Effects & blend modes XML: `decompiled_apk/assets/effects/`
    - Shapes XML: `decompiled_apk/assets/shapes/`
    - Translations/Strings: `decompiled_apk/res/values/strings.xml`
    - APK version: `decompiled_apk/apktool.yml`

- **SVG Generation:**
  - Shape previews are generated by running script blocks from shape XMLs inside a `vm` sandbox (see `scripts/lib/shape-renderer.ts`).

## Developer Workflow

Use **Bun** (do not use npm/pnpm/yarn).

- **Install dependencies:** `bun install`
- **Rebuild docs:** `bun run build` (runs generator and VitePress build)
- **Start dev server:** `bun run docs:dev`
